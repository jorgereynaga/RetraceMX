Necesito mejorar el módulo Caja del sistema Acopio360, utilizado para registrar pagos derivados de operaciones de compra de materiales en una empresa recicladora.

Actualmente la vista permite buscar una operación, ver el total, pendiente, seleccionar método de pago, capturar monto, referencia y observaciones. Sin embargo, se requiere mejorar la operación para que funcione como una caja real: calcular efectivo recibido, cambio a devolver, pagos parciales, pagos mixtos, cierre de operación, validaciones y ticket de pago.

Objetivo

Optimizar el módulo de caja para que el usuario pueda registrar pagos de manera rápida, segura y clara, evitando errores en montos, pagos duplicados, cierre incorrecto de operaciones o confusión entre importe total, pendiente, monto recibido y cambio.

3. Cálculo automático de cambio

Cuando el método de pago sea Efectivo, el sistema debe permitir capturar cuánto dinero se recibió físicamente y calcular automáticamente el cambio.

Reglas para pago en efectivo
El usuario captura el monto recibido.
El sistema calcula automáticamente:
Cambio = Monto recibido - Monto pendiente
Si el monto recibido es mayor al pendiente:
el pago aplicado debe ser igual al pendiente;
el sistema debe mostrar el cambio a devolver.
Si el monto recibido es igual al pendiente:
cambio igual a $0.00;
la operación queda pagada.
Si el monto recibido es menor al pendiente:
registrar como pago parcial;
la operación queda con estado “Pago parcial”.
Ejemplo
Total operación: $45,689.63
Pendiente: $45,689.63
Monto recibido: $46,000.00

Monto aplicado: $45,689.63
Cambio a devolver: $310.37
Estado: Pagado
Validaciones
No permitir monto recibido en cero para registrar pago.
No permitir montos negativos.
No permitir registrar un pago mayor al pendiente como “monto aplicado”.
En efectivo sí se puede recibir más dinero, pero el excedente debe calcularse como cambio, no como pago adicional.
El cambio debe resaltarse visualmente.

Actualmente existe un botón para cerrar operación. Debe tener reglas más estrictas.

Reglas
No permitir cerrar una operación con saldo pendiente, salvo que el usuario tenga permiso especial.
Si la operación tiene saldo pendiente, mostrar alerta:
La operación aún tiene un saldo pendiente de $25,689.63.
No puede cerrarse hasta liquidar el pago o registrar una autorización de cierre con saldo pendiente.
Si el saldo pendiente es cero:
permitir cerrar la operación;
cambiar estado a “Cerrada”;
bloquear nuevos pagos o partidas, salvo ajustes autorizados.
Cierre con saldo pendiente

Solo usuarios autorizados deben poder cerrar con saldo pendiente.

En ese caso, solicitar:

motivo;
observaciones;
usuario que autoriza;
fecha y hora;
saldo pendiente reconocido.

Reglas de cálculo
pagado_total = suma de pagos activos
pendiente = total_operacion - pagado_total

Para efectivo:

si monto_recibido >= pendiente:
    monto_aplicado = pendiente
    cambio = monto_recibido - pendiente
    estado = pagado
si monto_recibido < pendiente:
    monto_aplicado = monto_recibido
    cambio = 0
    estado = parcial

Para métodos distintos a efectivo:

monto_aplicado <= pendiente
cambio = 0

registrar pagos mixtos;
